# 發行說明

本頁會記錄Adobe Experience Manager Sites Optimizer的最新更新、新功能和改善專案。

## 2026年5月11至22日

### 新功能

- **網站警示報告** — 新的90天網站警示報告提供您網站健康狀況的季度檢視，使用以顏色編碼的每日區塊來強調提升警示期間，以便您快速識別及調查一段時間的趨勢。
- **操作遙測上線** — 尚未連線操作遙測資料的網站現在會在首頁上收到持續橫幅和引導式上線對話方塊，以完成設定，以確保您完全瞭解真實使用者效能。
- **替代文字：多網站管理員感知** — 為使用AEM多網站管理員或語言副本的網站產生替代文字修正時，Sites Optimizer現在會先檢查修正是否可以安全地套用至每個語言變體，然後再建議修正。

### 增強功能

- **替代文字正確性** — 替代文字建議現在從最近的稽核訊號中提取，並且重新偵測的問題會出現在目前問題和已部署的索引標籤中，以取得完整圖片。

### 錯誤修正

- 「部署」按鈕狀態現在會正確反映修正是否可實際部署。
- 深色主題現在可正確套用於頁面重新整理。
- 報表會顯示使用者地區設定的日期。
- 語言和數字/日期格式的區域偏好設定現在可以獨立設定。
- 熒幕助讀程式現在可存取損壞的影像替代文字。

## 2026年4月21日至5月10日

### 新功能

- **無網站已上線狀態** — 尚未新增網站的客戶，現在會在首頁上看到清晰、可操作的提示，以便快速開始使用。
- **說明中心檔案** — 現在可直接從應用程式內說明中心存取Experience League上的AEM Sites Optimizer檔案，無需離開產品。

### 錯誤修正

- 沒有作用中建議的網站現在會正確顯示「需要動作」對話方塊。
- 略過的建議現在會如預期顯示在「已忽略」標籤中。
- 付費流量選擇器下拉式清單不再截斷翻譯的文字。
- Sitemap頁面選擇器的大小現在已正確設定。

## 2026年3月13日至4月20日

### 新功能

- **試用上線** — 新的試用使用者現在會體驗引導式設定流程：進入您的網域、等待分析，然後探索您的第一個機會 — 無需設定即可開始。
- **試用機會頁面** — 試用版使用者可以搜尋、排序和篩選機會，其中三個建議已解除鎖定，其餘的建議會顯示在具有升級提示的鎖定預覽中。
- **每月最佳化進度** — 首頁上的進度列會追蹤您在本月執行了多少最佳化動作，協助您隨時掌控網站健康目標。
- **稽核目標URL** — 在「設定」下，您現在可以指定最多100個自訂URL，以確保這些頁面一律包含在稽核中。
- **傳遞型別設定** — 設定現在可讓您指定網站的傳遞型別（Edge Delivery Services、AEM Cloud Service或AEM Managed Services）並連線您的內容提供者。
- **Core Web Vitals重新設計** — Core Web Vitals機會已重新設計，包含Jira連結、CSV下載和批次動作的複選支援。
- **中斷的背景連結統一表格** — 來自所有來源的中斷背景連結現在會顯示在單一統一表格中，且可以直接匯出CDN重新導向規則。
- **沒有摺頁上的CTA：部署到作者** — 對沒有摺頁機會上的CTA的修正現在可以直接部署到AEM作者。
- **Forms Autofix Deploy** — Forms機會修正現在可以直接部署至AEM Author。
- **AEM多網站管理員支援** — 影響網站多個語言副本的機會現在會使用「修正位置」欄，指出修正套用到的根網站。
- **略過失敗的修正** — 您現在可以略過部署失敗的個別修正，讓您的工作流程保持解除封鎖。
- **在AEM編輯器中開啟** — 商機建議現在包含一個直接連結，以便在AEM的視覺編輯器中開啟受影響的頁面，以進行快速內嵌編輯。

## 2026年2月28日至3月13日

### 新功能

- **廣告意圖不符機會** — 新的機會型別可識別未轉換的付費流量登陸頁面、顯示跳出率、每次點按成本和流量量度，以協助您優先改善登陸頁面。
- **沒有CTA在摺疊上方** — 這個機會現在是專用的第一類型別，有自己的詳細資訊頁面和篩選，可以更輕鬆地追蹤和優先處理轉換改進。
- **Sitemap URL建議** — Sitemap機會現在會建議傳回404錯誤的頁面使用替代URL，以便更輕鬆修正損毀的Sitemap專案。
- **重新設計的無效連結** — 「無效連結」詳細資訊頁面已重新設計，以提高清晰度和可用性。

### 增強功能

- **熱門有機搜尋頁面V2** — 有機流量資料現在源自30天的Ahrefs資料集，提供更完整且可行的搜尋效能深入分析。
- **安全性漏洞：相依性樹狀結構** — 安全性漏洞詳細資料現在包含相依性樹狀結構視覺效果，因此您可以瞭解漏洞對專案的完整影響。

## 2026年2月14至27日

### 新功能

- **熱門有機搜尋頁面** — 網站運作狀態監視現在包含一個專用索引標籤，顯示您網站熱門的有機流量頁面，讓您瞭解哪些內容驅動了最多的搜尋流量。
- **Alt Text Autofix V2** — 部署Alt Text修正之前，您可以執行預檢「檢查修正能力」評估，以確認修正可成功套用至您的內容。
- **已部署的替代文字檢視** — 替代文字修正現在出現在已部署的標籤中，為您提供協助工具改善的完整歷史記錄以及目前未解決的問題。
- **外部組織部署閘道** — 將修正部署至外部管理的網站時，現在需要明確的確認步驟，以防止意外變更。

### 增強功能

- **Meta標籤URL劐免** — 現在可以透過設定從Meta標籤驗證中排除特定URL，減少刻意簡短或非標準標題的誤判。
- **進階URL篩選** — 商機清單現在支援依URL篩選時的子路由首碼比對，讓您更容易聚焦於網站的特定區段。
- **改進的趨勢圖** — 流量趨勢圖現在可以正確處理逐年比較資料，移除年邊界誤導性的下降。

## 2026年2月6至13日

### 新功能

- **維護模式** — Sites Optimizer現在會適當地處理計畫的維護期間，顯示清楚的狀態訊息，而不是停機期間不完整或誤導的資料。
- **中斷回溯連結的已部署檢視** — 現在會在「已部署」索引標籤中追蹤已修正回溯連結，並依日期分組，以便您一眼就能看到修正歷史記錄。
- **在摺頁機會上方沒有CTA** — 新的機會型別會出現在摺頁上方看不到清晰call-to-action的頁面，可幫助您識別和改善轉換潛力低的頁面。
- **協助工具與色彩對比的Jira整合** — 現在，Forms和色彩對比協助工具機會可以直接連結至Jira票證，以便在現有工作流程中簡化問題追蹤。

### 增強功能

- **Meta Tags &amp; Security的已部署檢視** — Meta Tags &amp; Security機會現在包含日期群組的已部署標籤，與其他機會型別一致。
- **替代文字部署追蹤** — 「標示為已部署」現在可用於替代文字修正，而且手動編輯的替代文字會在重新分析執行時保留。

## 2026年1月26日至2月6日

### 新功能

- **Canonical和Hreflang的部署檢視** — Canonical和Hreflang機會的變更現在會依部署日期分組在「已部署」索引標籤中，讓您清楚瞭解修正內容和修正時間的歷史記錄。
- **CSV匯出** — 您現在可以將High Organic Low CTR和Forms機會的機會資料匯出至CSV，以進行離線分析和報告。
- **我的最愛商機** — 從標題開始任何商機，將其新增至您的最愛，讓您更快導覽回您正在積極工作的商機。
- **重新導向鏈結的已部署檢視** — 現在可以直接從詳細資訊頁面將重新導向鏈結修正標籤為已部署。

### 增強功能

- **已改善的Cookie橫幅成本預估** — Cookie橫幅機會的成本計算已更精確。

## 2026年1月16至23日

### 新功能

- **網站健康狀態監視（一般可用性）** — 網站健康狀態監視現在可供所有客戶使用，提供您網站效能健康狀態的持續檢視。 新網站會在上線時自動設定。
- **子路徑網站支援** — 網站健康情況監視現在完全支援範圍為特定URL子路徑的網站。

### 增強功能

- **可操作的資料充足性通知** — 少於1,000個頁面檢視的付費流量機會現在會顯示資料充足性通知，協助您將流量資料在統計上有意義的最佳化工作集中在一起。
- **彈性的Meta標題驗證** — 降低中繼標題的最低字元需求，讓您在製作簡潔的頁面標題時擁有更多彈性。
- **本地化新增功能對話方塊** — 應用程式內功能公告對話方塊現在會以您偏好的語言顯示。
- **已發佈的徽章** — 已部署的High Organic Low CTR機會中的變化現在會顯示「已發佈」徽章，以便更輕鬆地區分作用中變更與待處理變更。
- **協助工具中的提取要求連結** — 協助工具機會的[已部署]索引標籤現在會顯示每個修正的相關提取要求URL，讓您更輕鬆地追蹤回原始檔控制歷程記錄中的變更。
